What is Azure?

Microsoft’s cloud platform providing:

  • Storage
  • Compute
  • Databases
  • DevOps: automated processes
  • AI & Machine Learning
  • IoT

What’s supported on Azure

Many languages and apps, but specifically

  • Microsoft stuff, unsurprisingly
    • .NET
    • .NET Core
    • C#
    • F#
    • TypeScript
  • Other languages (first class)
    • Java
    • JavaScript
    • Python
    • Go
    • PHP
    • Powershell
    • Ruby

Azure jargon

Regions: “a set of datacenters deployed within a latency-defined perimeter and connected through a dedicated regional low-latency network”

Geographies: Regions bound by similar criteria, often legal

Availability Zones: Distinct zones within a region to tolerate local failures

Virtual machines

  • What’s a virtual machine vs app service?
  • Why would I use a virtual machine?
    • Very customizable
    • High level of control
    • “Out of the box” solutions don’t work for your needs

App service

  • Platform as a service
  • Lower maintenance than building out your own VM
  • Many “standard” builds
  • You can submit your own Docker container to be managed
  • Will you use this or VMs in the Capstone?

App service automation

  • Provisioning
  • Deployment (Git, GitHub, etc)
  • Load balancing
  • Scaling
  • Backup

Scaling terminology

  • Vertical: Instances get beefier, but you don’t create more of them
    • Level up your character
  • Horizontal: More instances, but same stats
    • More people in your raiding party
  • Usually there’s a combination, though not always

Azure Compute Units

  • Azure has a baseline VM instance, Small VM
    • Also known as Standard_A1
  • The performance offered by this VM is set at 100 Azure Compute Units (ACU)
  • Other VMs have more or (rarely) less ACU, and are priced accordingly
  • https://docs.microsoft.com/en-us/azure/virtual-machines/acu
