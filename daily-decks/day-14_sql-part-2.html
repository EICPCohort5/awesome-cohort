Database Definition Language (DDL)

  • CREATE TABLE [tablename] ( field specifications, constraint specifications )
  • Field specification: [fieldname] [data type] [constraints]
    • Constraints not null, auto_increment, etc.
  • Constraint specification: constraint [name] [constraint]
    • Primary keys
    • Foreign keys
    • Check values
    • And more!

Keys

  • Primary keys
    • Can have multiple fields
    • Often single fields
    • Good practice: Arbitrary field, with no actual significance
  • Foreign keys
    • Reference another table
    • Allow you to set referential integrity

Altering existing tables

  • Not necessary in early dev, just re-build the tables with new specs
  • Needed when data is in production and you can’t drop / rebuild the table
  • ALTER TABLE [changes]
    • Can include an ALTER COLUMN subclause to change a particular column

Joins

Joining two tables together.

Why do we have separate tables?

Types of join:

  • Cross join: no one actually uses this, ignore it
  • inner join: exact match between tables, no nulls
  • outer join: deals with non-matching queries, or nulls in one table (usually the second)
    • full outer join: all rows all tables, not supported in mySQL, let’s not worry about it
    • left outer join: all rows from the first (left-hand) table, only matches from the second
    • right outer join: matching rows in the first table, all matches from the second (right-hand) table
  • self joins: good for hierarchies

Inner Join syntax

Assume the genres table has genre (the name) and genre_id (the primary key)
Assume the movies table has a genre_id field that’s a foreign key to genres

SELECT m.title, g.genre 
FROM movies 
  inner join genres g
  on m.genre_id = g.genre_id

Order By

Orders the result set according to the criteria specified in the result set

SELECT s.student_id, s.first_name, s.last_name, s.country
FROM students s
ORDER BY s.last_name;

Set functions

Also known as aggregate functions

count (nulls with *) max (no nulls) min (no nulls) avg (no nulls) sum (no nulls)

Set function demos

-- COUNT
SELECT COUNT(*) uk_students FROM students s
WHERE country = 'UK';

-- MAX
SELECT MAX(start_date) latest_start_date FROM students s;

-- MIN
SELECT MIN(start_date) earliest_start_date FROM students s;

-- MAX / MIN
SELECT MAX(start_date) latest_start_date, MIN(start_date) earliest_start_date FROM students s;

Group By

Usually used with set functions Any non-aggregate columns should be grouped by (group byed?)

SELECT COUNT(*) student_count, country FROM students s
GROUP BY country;

Having

A WHERE clause for aggregate data, or against the group by Also note the cool stuff you can do with aliasing

select count(*) as student_count, 
         count(s.github_id) as github_count, 
       count(*) - count(s.github_id) as difference,
       s.country 
from students s
group by country
having difference > 0;
