Designing a RESTful API

  • Nouns not verbs
  • Nouns / Resources / Endpoints
  • Query strings (/books?author=Mellville) as modifiers, not required
  • Expand the URI for utilities
    • /books/123
    • /cars/{vin}/register
  • HTTP verbs for low-level actions

Versioning Strategies

  • Query string? (Really?)
  • In the URI (useful, used often)
  • Arbitrary header (easy to forget, need a fallback)
  • Content-type: application/vnd.whatever.v1
